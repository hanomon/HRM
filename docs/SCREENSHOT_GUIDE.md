# 📸 스크린샷 가이드

## 필요한 스크린샷

아래 스크린샷들을 촬영하여 `docs/images/` 폴더에 저장해주세요.

### 1. dashboard.png - 대시보드 메인 화면
**촬영 방법:**
1. 브라우저에서 http://localhost:5173 접속
2. 테스트 데이터가 있는 상태에서 대시보드 화면 열기
3. 출퇴근 기록이 보이는 전체 화면 캡처

**포함되어야 할 요소:**
- 상단 네비게이션 메뉴
- 출퇴근 기록 테이블
- 날짜 필터
- Excel 내보내기 버튼
- 최소 3~5개의 기록이 보이도록

---

### 2. nfc-tagging.png - NFC 태깅 대기 화면
**촬영 방법:**
1. "NFC 태깅" 메뉴 클릭
2. "NFC 스캔 시작" 버튼 클릭 후 대기 화면
3. "카드를 가까이 대주세요" 메시지가 보이는 화면 캡처

**포함되어야 할 요소:**
- NFC 아이콘/애니메이션
- 안내 메시지
- 현재 상태 표시

### 3. employees.png - 직원 관리 화면
**촬영 방법:**
1. "직원 관리" 메뉴 클릭
2. 직원 목록이 보이는 화면 캡처

**포함되어야 할 요소:**
- 직원 목록 테이블 (이름, 부서, 직책, NFC ID)
- "직원 추가" 버튼

### 4. add-employee.png - 직원 추가 모달/폼
**촬영 방법:**
1. "직원 관리" 화면에서 "직원 추가" 버튼 클릭
2. 입력 폼이 열린 화면 캡처 (입력 예시 포함)

**포함되어야 할 요소:**
- 이름 입력 필드
- NFC ID 입력 필드
- 부서 입력 필드
- 직책 입력 필드
- 이메일 입력 필드
- 전화번호 입력 필드
- 저장/취소 버튼

**입력 예시:**
- 이름: 홍길동
- NFC ID: 04:D1:E2:F3:A4:B5:C6
- 부서: 영업팀
- 직책: 과장
- 이메일: hong@company.com
- 전화번호: 010-9876-5432

### 6. tag-success.png - 태깅 성공 화면
**촬영 방법:**
1. NFC 태깅 후 성공 메시지가 표시된 화면
2. 직원 정보와 출근/퇴근 정보가 보이는 화면 캡처

**포함되어야 할 요소:**
- 성공 아이콘 (✅)
- 직원 이름
- 출근 또는 퇴근 표시
- 태깅 시간
- 환영 메시지 또는 인사 메시지

**시뮬레이션 방법 (실제 NFC 카드 없이):**
```javascript
// 브라우저 콘솔에서 실행
fetch('http://localhost:3000/api/attendance', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ nfc_id: '04:A1:B2:C3:D4:E5:F6' })
})
.then(res => res.json())
.then(data => {
  console.log('태깅 성공:', data);
  alert(`${data.employee_name}님 ${data.tag_type === 'check_in' ? '출근' : '퇴근'} 완료!`);
});
```

## 스크린샷 촬영 팁

### 권장 사항
1. **해상도**: 1920x1080 이상
2. **브라우저**: Chrome 또는 Edge (개발자 도구 F12 -> 디바이스 시뮬레이터 사용 가능)
3. **확대/축소**: 100% (Ctrl+0)
4. **테마**: 라이트 모드 권장 (문서에서 보기 좋음)
5. **더미 데이터**: 테스트 데이터를 사용하여 현실적인 화면 구성

### 촬영 도구
- **Windows**: `Win + Shift + S` (Snipping Tool)
- **Mac**: `Cmd + Shift + 4`
- **Chrome 확장**: Awesome Screenshot, Nimbus Screenshot
- **전체 페이지**: Chrome DevTools (F12) -> `Ctrl + Shift + P` -> "Capture full size screenshot"

### 편집 (선택사항)
- 민감한 정보 블러 처리
- 중요한 버튼/영역 화살표로 표시
- 테두리 추가로 화면 구분 명확하게

### 파일명 규칙
- 소문자 사용
- 단어는 하이픈(-)으로 구분
- 명확하고 설명적인 이름
- 예: `dashboard.png`, `add-employee.png`

### 파일 크기
- PNG 형식 권장
- 1MB 이하로 최적화 (TinyPNG 등 사용)

## 스크린샷 추가 후 확인

README.md 파일을 열어서 이미지가 제대로 표시되는지 확인:
```bash
# VS Code Preview 또는 GitHub에서 확인
```

이미지가 보이지 않으면:
1. 파일명이 정확한지 확인
2. 파일 경로가 맞는지 확인 (`docs/images/파일명.png`)
3. 파일 확장자가 소문자인지 확인 (.PNG가 아닌 .png)
